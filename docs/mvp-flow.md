# PakuFit MVPフロー

## 全体画面フロー

1. 起動
2. ログイン / 新規登録
3. 目標設定（初回のみ）
4. 食事登録トップ
   - 写真アップロード
   - テキスト入力
5. AI推定待機/表示
6. ユーザー補正
7. 食事確定
8. 1日サマリー
9. 次の食事候補提案
10. 安全表示

## 食事写真アップロード

- 写真1件を「食事記録」へ紐付け
- 撮影日時/登録日時を保存
- 画像は圧縮・メタデータを最小化して保存（位置情報など必要時のみ）
- OCR/タグは将来拡張（MVPではUI受領が先行）

## AI推定

1. 食事画像を解析対象としてキュー投入（MVP設計上のイベント）
2. 料理名候補、主な食材、調理方法候補、量の初期推定、信頼度を受け取る想定
3. 推定結果をカード表示
4. 信頼度が低い場合は補正を強制的に提示

### 推定UI要件

- 料理名候補: 上位3〜5
- 食材候補: 主要食材
- 量: 標準サイズを初期値として提示
- 信頼区分: 高/中/低
- 注意文: 「推定値は概算です。実食事と違う場合は修正してください」

## ユーザー補正

- 料理名・量・食べた割合・調理方法の項目を編集可能
- 料理名自由入力
- 量はg/皿/杯など複数単位を許容
- 食べた割合は0〜100%のスライダーと固定ラベル
- 修正結果は確定前に再計算して差分を表示

## カロリー/PFC概算

- AI推定値をベースに、補正後値を再計算
- 料理別表示: 推定値・補正後値
- 合計表示: 日計（暫定）
- 表記ルール: 「目安」「概算」「推定」と明示

## 1日サマリー

- 日付、食事回数、記録済み件数を表示
- 総合計
  - カロリー
  - たんぱく質
  - 脂質
  - 炭水化物
- 残り目安（目標差）
  - カロリー残
  - PFC残量（不足/超過）

## 次の食事候補提案

- 画面表示タイミング: 食事登録後、またはサマリ閲覧時
- 入力条件
  - 時間帯（朝/昼/夜/夜食）
  - 残りカロリー
  - 不足PFC
  - 食事履歴
  - 利用希望（コンビニ/自炊/外食）
  - 予算
- 出力
  - 候補名（例: 具体的な商品名ではなく「候補カテゴリ/料理例」優先）
  - 1食あたり目安
  - 不足PFCを補いやすい理由
  - 注意喚起ラベル
  - 候補としての扱い明示

## 安全表示

- 健康効果断定を含まない説明文
- 補食例の際に「食事の傾向」を示すが「確定アドバイス」はしない
- 持病/妊娠中/未成年/摂食障害リスク情報の入力時は、医療相談の案内を表示

## 画面遷移例（簡易）

- 記録画面 → 推定表示 → 補正画面 → 確定 → サマリ画面 → 候補提案
