# 写真AI概算のプライバシー文言（草案 / DRAFT）

> ステータス: **草案**。最終的な「外部送信の可否・既定運用・保持方針・同意方式」はユーザー（および必要なら法務）が決定する。
> 本ドキュメントは実装済みの挙動の整理と、決定が必要な論点・文言案のたたき台であり、確定方針ではない。
> live の `app/privacy/page.tsx` はこの草案では変更しない。

## 1. 現状の実装挙動（事実）

- 写真AI概算は **任意機能**。ユーザーが `/meals/new` で写真を選び「AIで概算する」を押したときのみ実行される。
- プロバイダは `PAKUFIT_VISION_PROVIDER` と APIキーで決まる（`lib/ai/estimate-meal-nutrition.ts`）:
  - `mock`（既定・キー無し）: **外部送信なし**。画像バイトから決定的な擬似値を生成するだけ。
  - `openai`: 画像を **OpenAI API**（Chat Completions vision, 既定 `gpt-4o-mini`）へ送信して概算。
  - `anthropic`: 画像を **Anthropic API**（Messages vision）へ送信して概算。
- 送信内容: 画像バイト（base64）＋食事名ヒント（任意）＋固定プロンプト。サーバ側（`app/api/meals/estimate`）から送信し、ブラウザから直接外部へは送らない。
- 概算結果（栄養値・ラベル・信頼度）と、補正後の最終保存値は `photo_estimate_logs` に本人のみ(RLS)で記録される（補正率観測用）。**画像そのものは保存していない**（現状、画像の永続保存は未実装）。

## 2. 決定が必要な論点（ユーザー判断）

1. **外部送信の既定可否**: OpenAI/Anthropic への画像送信を「既定ON（キー設定で自動）」にするか、「明示オプトインONのときだけ」にするか。
   - 現状はキーがあれば自動的に実プロバイダが使われる（オプトインUIは無い）。
2. **同意取得の方式**: 初回利用時の同意ダイアログ／設定でのオプトイン／プライバシーポリシー同意で足りるとするか。
3. **送信先プロバイダの開示**: ユーザーに「どの事業者へ送るか（OpenAI 等）」を明示するか。
4. **保持・再利用方針**: 送信先での画像の保持期間・学習利用の有無について、各プロバイダの最新規約に基づきどう説明するか（例: API 経由は学習に使われない等の確認が必要）。
5. **未成年・センシティブ配慮**: 食事写真に人物・周辺情報が写り込む可能性への注意喚起。

## 3. ユーザー向け注意文（文言案・JA）

> 確定前提ではない。決定後に `/meals/new` のAI概算ブロックや `/privacy` へ反映する想定。

- 概算ブロック近傍（送信前の説明）案:
  - 「AI概算を使うと、選んだ料理写真が栄養推定のため外部のAIサービス（例: OpenAI）に送信されます。送信は概算のときだけで、画像はアプリには保存されません。値は概算のため、保存前に確認・補正してください。」
- 設定/プライバシー記載 案:
  - 「写真AI概算は任意機能です。利用時のみ、料理写真を概算目的で外部AI事業者に送信します。送信した画像をアプリ側で保存・再利用することはありません。AI概算は医療・診断を目的とせず、結果は『概算/目安』です。」
- オプトインにする場合のトグル文言 案:
  - 「写真からのAI概算を有効にする（料理写真を外部AIサービスへ送信します）」

## 4. 既存ポリシーとの整合（`docs/privacy-and-safety.md`）

- 「画像は用途外で再利用しない」「概算と明記」「医療目的としない」既存方針と矛盾しない範囲で記載する。
- 既存方針の「集計処理は匿名加工」に合わせ、`photo_estimate_logs` の集計（`scripts/estimate-accuracy.sql`）はプロバイダ別の統計のみで、個人を特定しない用途に限定する。
