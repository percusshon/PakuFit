# 開発ロードマップ

## Phase 0: docs/設計

- 本リポジトリを設計ドキュメントのみで構成
- MVP要件、セーフティ、RLS、DB、提携方針を定義

## Phase 1: Next.js + Supabase scaffold

- 認証基盤の雛形
- プロジェクト構成、CI基盤、環境変数設計
- 画像保存と日次集計の基本テーブル整備（設計通り）

## Phase 2: 食事記録MVP

- 写真/テキスト入力画面
- 食事一覧・日次一覧
- 目標登録画面（体重管理/バランス重視/高たんぱくなど）

## Phase 3: カロリー/PFC概算

- 推定値を概算表示
- 補正UIの実装
- 日次サマリーと残量表示

## Phase 4: 次の食事候補提案

- ルールベース候補生成を導入
- 候補理由表示と安全ラベル
- 「候補」表現を徹底

## Phase 5: 写真AI連携

> 状態: 概算機能の基盤を実装済み（モック既定 + Anthropic vision 差し替え可、commit 438eaf5）。実プロバイダの本確定（モデル/コスト/プライバシー方針）は未着手。

- 画像解析サービス接続（設計外部API）
- 推定精度ログと補正率の観測
- 品質が低いケースで補正導線最適化

## Phase 6: バーコード/JANコード

- 条件分岐で手入力/バーコード入力を共存
- JAN一致時の候補自動補完

## Phase 7: コンビニ商品DB

- 商品DB同期、在庫・価格・地域条件の導入
- 提携先情報の表示分離（広告表示と非広告表示）

## Phase 8: 提携/広告/収益化

- 説明付き広告導線の追加
- 利用者透明性（提携表示、広告表示）を担保

## Phase 9: モバイルアプリ化

> 状態: PWA 基盤を実装済み（manifest/icon/SW・インストール可能化・オフラインフォールバック、commit 3f2e496）。カメラ連携最適化・高度なオフライン対応は未着手。

- PWA/ネイティブ移行検討
- カメラ連携最適化
- オフライン考慮

## 進行指標（共通）

- ユーザー補正率と、推定利用満足度の観測
- 安全表示の表示率（必要時に表示されているか）
- 相談誘導文言の適切率
