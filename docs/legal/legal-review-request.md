## 0. 依頼の概要

食事記録サポートアプリ「PakuFit（パクフィット）」を一般公開するにあたり、
以下のドラフト文書の適法性・十分性のレビューと、末尾「6. 質問リスト」への助言をお願いしたい。

- プライバシーポリシー（`privacy-policy-draft.md`）
- 利用規約（`terms-of-service-draft.md`）
- 特定商取引法に基づく表記（`tokushoho-draft.md`／将来の有料化時）

あわせて事実整理として `personal-data-inventory.md`（個人データ取扱台帳）を添付する。

## 1. サービス概要

- 種別: 食事記録サポートのWebアプリ（PWA／スマホ利用想定）。Next.js + Supabase。
- 目的: 写真・テキストから食事を記録し、カロリー/PFC（たんぱく質・脂質・炭水化物）を**概算**表示。次の食事候補をルールベースで提示。
- 想定ユーザー: 記録が続きにくい成人。コンビニ/外食中心の層。
- **非医療**: 診断・治療・効果保証を目的とせず、全栄養値を「概算・目安」として提示。極端な減量・断食誘導をしない方針。
- 現段階: **MVP・ローカル開発段階**。本番公開・課金・広告配信は未実施。

## 2. ビジネスモデル（現状と将来）

- 現状: 無償提供。課金・広告なし。
- 将来構想（`partner-product-roadmap.md`）: コンビニ/スーパー商品DB連携、提携・広告/アフィリエイト導線（透明性・ステマ対策前提）、有料機能の可能性。
  - これらは未実装で、導入時に別途レビューを想定。

## 3. 取得データとデータフロー

詳細は `personal-data-inventory.md`。要点:

- メール（認証）、プロフィール、食事記録、食事写真、健康目標、概算栄養値、提案履歴、各種ログを本人単位で保管（RLSで本人のみアクセス）。
- 外部送信は2系統:
  1. **Supabase**（認証・DB・画像保管）= サービス基盤。
  2. **外部AIサービスへの写真送信** = **明示オプトイン同意時のみ**。料理画像＋食事名ヒストを概算目的でサーバから送信。アプリ側で画像は保存・再利用しない。
- 解析/広告タグ・第三者Cookieは現状未導入。

## 4. セーフティ設計（表現規制対応の現状）

- 「痩せる」「治療」「効果がある」等の断定/結果保証表現を出力に混入させない方針。推薦ロジックのテストで機械的に検査。
- 推定値は全画面で「概算/目安/要確認」と明示。
- 未成年・妊娠中・通院中・摂食に不安のある利用者へ注意文表示、専門家相談導線。
- 詳細: `../privacy-and-safety.md`、`/safety` ページ。

## 5. 想定される適用法令（開発側の暫定整理・要検証）

| 法令 | 関係する論点 |
| --- | --- |
| 個人情報保護法（APPI） | 利用目的の特定・公表、第三者提供（27条）、**外国にある第三者への提供（28条）**＝海外AI事業者・海外リージョン、安全管理措置（23条）、保有個人データの開示・訂正・利用停止（32-35条）、苦情窓口 |
| 電気通信事業法 外部送信規律（27条の12） | ブラウザからの利用者情報の外部送信に対する通知・公表義務。現状は解析/広告タグ未導入の想定。該当範囲の確認 |
| 特定商取引法 | 有料・サブスク提供時の表記義務（現状無償、将来テンプレを用意） |
| 景品表示法（含むステマ規制 令和5年告示） | 将来の提携・広告表示、ステルスマーケティング対策 |
| 健康増進法・食品表示法 | 栄養・健康に関する誇大表示の禁止 |
| 医薬品医療機器等法（薬機法） | 効能効果の標榜規制（非医療の徹底） |
| 消費者契約法 | 利用規約の免責条項の有効性（全部免責条項の無効リスク） |
| 民法・青少年保護 | 未成年者の契約・同意（成年年齢18歳） |

## 6. 質問リスト（弁護士への確認事項）

### 個人情報保護法
1. 外部AIサービス（海外想定）への画像送信は、**委託**・**第三者提供**・**外国にある第三者への提供（28条）**のいずれで整理すべきか。本人同意取得の文言・粒度は現行オプトインで足りるか。
2. Supabase の保管リージョンが海外の場合、28条対応として何が必要か（同意 / 基準適合体制 / 提供先国の情報提供）。
3. `user_goals.notes` 等の自由記述に**要配慮個人情報**（病歴等）が混入した場合の取扱い。事前同意やフィルタの要否。
4. 食事写真への**第三者の顔・所在情報の写り込み**への対応は、注意喚起で足りるか。
5. プライバシーポリシーの**利用目的**の記載粒度・**保有個人データに関する事項**（開示等請求手続・窓口）の十分性。

### 利用規約・消費者法
6. 免責条項（概算値・非医療・結果非保証）の有効な書き方。消費者契約法上、全部免責とならない表現。
7. 準拠法・**裁判管轄**条項（消費者契約での専属的合意管轄の有効性）。
8. **未成年**の利用可否・親権者同意の要否と取得方法。

### 表現・広告
9. 栄養・健康表示が**健康増進法/薬機法/景表法**に抵触しない表現ラインの確認。
10. 将来の提携・広告導線における**ステマ規制**対応（表示区分・ラベルの十分性）。

### 外部送信規律
11. 現状（解析/広告タグ未導入、AI送信はサーバ側）で**電気通信事業法 外部送信規律**の通知・公表義務の該当有無。将来タグ導入時の対応。
