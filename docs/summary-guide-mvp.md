# PakuFit サマリー読み取りガイドMVP（Phase 8）

## 実装範囲

- `/summary` に「目標に合わせた読み取りガイド」を追加。
- `getTodayMealSummary` と `getCurrentUserGoal` の取得値を使って、画面上にガイドカードとして表示。
- ガイドは診断や治療ではなく、`食事記録をサポート`する読み取り補助情報として扱う。
- DBの追加変更は行わず、既存の `meal_entries` / `nutrition_estimates` / `user_goals` を参照。

## 目標別読み取りガイド

- `higher_protein`
  - たんぱく質入力があると、次の候補を参考にしやすい。
- `lower_fat`
  - 脂質の概算入力があると、脂質控えめの候補表示へつなぎやすい。
- `balanced_meals`
  - カロリーだけでなくPFCの確認があると、記録を見返しやすい。
- `convenience_store_friendly`
  - 外食/コンビニ中心の日も、カテゴリ単位で振り返りやすい形にする。
- `weight_management`
  - 日々の記録を続ける観点で件数と概算値を確認する。
- 目標未設定
  - `/settings/goals` へ導線を出し、目標別表示を有効化。

## 入力データ

- 当日食事件数
- 当日の概算カロリー合計
- 当日の概算たんぱく質 / 脂質 / 炭水化物合計
- PFC入力件数（`nutrition_estimates`）
- 現在の `user_goals`（未設定可）

## 出力データ

- 読み取りガイドカード配列
- データ充足度（0〜1 と `low/medium/high`）
- 注意文（PFC入力不足時は限定性を明示）
- 目標未設定フラグ

## 禁止表現

- 使用しない表現:
  - `食べるべき`, `痩せる`, `減量できる`, `脂肪が落ちる`, `血糖値が改善`, `病気を防ぐ`, `治療`, `診断`, `疾病改善`, `AI栄養指導`

## 未実装範囲

- 医学的判定や体重推定、治療的な助言
- コンビニ商品名の個別提案
- 写真解析・バーコード連携・広告提案

## 今回の方針

- 表示は「目標に合わせた読み取りガイド」としてのみ提示し、強い断定や推奨を避ける。
- 候補と同様に、あくまで一般的な食事管理の参考情報として扱う。
