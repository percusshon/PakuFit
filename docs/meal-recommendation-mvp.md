# PakuFit 次の食事候補MVP（Phase 5）

## 実装範囲

- `/recommendations` で本人の当日サマリーを取得して候補を表示
- AIを使わず、固定ルールのみで次の食事候補を生成
- 今日の記録がない場合は、まず記録を保存する導線を提示
- PFC入力が少ない場合は、データ不確実性を明示
- 候補は候補カテゴリ + 理由 + 注意文をカード形式で表示
- ユーザーIDは server 側認証ユーザー由来で扱い、クライアントから受け取らない

## 運用方針

- `meal_recommendations` への永続保存は今回見送り（表示優先）
- 将来必要になった場合のみ、`meal_recommendations` に「推奨候補履歴」を保存予定
- サマリー取得は既存 `getTodayMealSummary` を拡張（PFC入力件数を取得）

## 固定ルールの前提

- まずは説明可能な簡易ロジックでMVPを回す
- 目標は「断定」ではなく「次の食事候補の提示」
- 候補はカテゴリ（たんぱく質/脂質/炭水化物/時間帯）中心

## テスト方針（Phase 5.5）

- テスト対象: `lib/recommendations/calculate-meal-recommendations.ts`
- カバレッジ目標: 固定ルールの主要分岐
  - 記録0件
  - PFC未入力が多い
  - たんぱく質が少ない
  - 脂質が高い
  - 炭水化物が少ない
  - 夕方以降
  - 禁止表現の出力有無
- `npm run test` で `vitest run` を実行
- 禁止表現を含む文言が出力されないことを CI 的に担保

## 推薦条件（現行）

- 今日の食事記録が0件
  - 「まず1件記録」を案内
- PFC入力未入力が多い
  - 不確実性文言を明示
- たんぱく質が少なそう
  - たんぱく質を足しやすい候補を追加
- 脂質が高め
  - 脂質を控えめにしやすい候補を追加
- 炭水化物が少なそう
  - 主食を足しやすい候補を追加
- 夕方〜夜間
  - 重なりを避けやすい候補を追加

## 禁止表現

- 次のような断定的・治療的表現は使わない
  - 「この食事で痩せる」
  - 「脂肪が落ちる」
  - 「病気を防ぐ」
  - 「食べるべき」
  - 「AI栄養指導」
  - 「治療」「診断」「疾病改善」

## 安全性の表示

- 「固定ルールによる参考候補」「AI推薦ではない」
- 「体調や持病、通院中の方は相談」を必ず併記
- 「極端な制限を促さない」方針を維持

## 将来拡張（未実装）

- `meal_recommendations` の保存/履歴表示
- パーソナライズ強化、時間帯・予算・在庫/希望チャネル対応
- コンビニ/外食候補の安全なカテゴリ提示（現フェーズは具体的商品名を提示しない）

## Phase 6（目標連携）

- `user_goals` を `getCurrentUserGoal` で取得し、`/recommendations` の候補作成に反映。
- 目標別の文言/優先度調整
  - `weight_management`: 記録継続を重視する文言
  - `balanced_meals`: バランスを取りやすい候補への寄せ
  - `higher_protein`: たんぱく質候補を優先
  - `lower_fat`: 脂質控えめ候補を優先
  - `convenience_store_friendly`: 外食/コンビニでも選びやすいカテゴリ候補を優先
- 目標未設定時は安全な案内を表示し、目標保存導線へ誘導。
